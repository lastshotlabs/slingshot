Once the normal package-first and entity-first paths stop fitting, this is where to go next.

This section is intentionally not the starting point.

Use it when:

- you need manual adapter wiring
- you need a raw plugin instead of a package
- you need to override generated entity behavior
- you need a supported alternate path like manifests
- you are debugging framework seams instead of using the normal surfaces

## The progression

1. Start with [Quick Start](/slingshot/quick-start/)
2. Learn the [Core Features](/slingshot/core-features/)
3. Learn the [Authoring Model](/slingshot/authoring-model/)
4. Use advanced docs only when the normal path is no longer enough
