Generated routes are the normal path. Overrides and extra routes are the main customization seams.

The examples on this page continue the [starter app](/slingshot/app-authoring/starter-app/) and
its `Post` entity module.

## What you get by default

When an entity has route config, Slingshot can generate:

- `create`
- `list`
- `get`
- `update`
- `delete`
- named-operation routes

That is the baseline behavior.

For the starter app's `Post` entity, that usually means routes like:

- `POST /blog/posts`
- `GET /blog/posts`
- `GET /blog/posts/:id`
- `PATCH /blog/posts/:id`
- `DELETE /blog/posts/:id`
- `GET /blog/posts/by-author/:authorId`
- `GET /blog/posts/by-slug/:slug`
- `POST /blog/posts/:id/publish`

## Override a generated route

Use overrides when you want to keep the generated route identity but replace its executor or docs
metadata.

```typescript title="src/blog/package.ts"
import { defineEntityExecutor, entity } from '@lastshotlabs/slingshot-entity';

const PostModule = entity({
  config: Post,
  operations: PostOperations,
  overrides: {
    get: defineEntityExecutor({
      summary: 'Get a post with viewer metadata',
      build:
        ({ entityAdapter }) =>
        async ({ params, respond }) => {
          const post = await entityAdapter.get(params.id);
          return respond.json({ post, viewerCanEdit: true });
        },
    }),
  },
});
```

## Add an extra route

Use an extra route when the route should live beside the entity shell but is not one of the
generated defaults.

```typescript title="src/blog/package.ts"
import { defineEntityRoute, entity } from '@lastshotlabs/slingshot-entity';

const PostModule = entity({
  config: Post,
  extraRoutes: [
    defineEntityRoute({
      key: 'preview',
      method: 'post',
      path: '/:id/preview',
      auth: 'userAuth',
      buildExecutor:
        ({ entityAdapter }) =>
        async ({ params, respond }) => {
          const post = await entityAdapter.get(params.id);
          return respond.json({ id: params.id, preview: post?.body?.slice(0, 140) });
        },
    }),
  ],
});
```

## Collision rules

Extra routes cannot silently collide with generated routes.

If you want to replace a generated route, use an override. Do not create an extra route that
matches the same method and path and hope the framework picks the right one.

That distinction is important enough that the runtime throws on invalid collisions.

## When to use what

Use:

- generated routes for the default case
- overrides when the route identity should stay the same
- extra routes when you want a new route inside the entity shell
