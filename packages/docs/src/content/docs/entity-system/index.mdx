Every backend app needs CRUD routes, input validation, auth checks, and database queries for its
data models. Writing all of that by hand is repetitive and error-prone. The entity system generates
it from a single definition.

Define your data shape. The framework generates:

- `GET /posts`, `GET /posts/:id`, `POST /posts`, `PATCH /posts/:id`, `DELETE /posts/:id`
- Input validation and OpenAPI docs
- Named operations like "publish" or "lookup by slug"
- Auth, permissions, and rate limiting per route
- Storage that works with memory, SQLite, Postgres, MongoDB, or Redis

You still own the data model. The framework just writes the boilerplate.

## What you'll learn

- How to define an entity with fields, indexes, and route defaults
- How to add operations beyond CRUD (lookups, transitions, aggregates)
- How to control auth, permissions, and middleware per route
- How to swap storage backends without changing your entity code
- How to override or extend generated routes when you need custom behavior

## A minimal entity

```typescript
import { defineEntity, definePackage, entity, field } from '@lastshotlabs/slingshot';

const Post = defineEntity('Post', {
  namespace: 'blog',
  fields: {
    id: field.string({ primary: true, default: 'uuid' }),
    title: field.string(),
    body: field.string(),
  },
});

export const blogPackage = definePackage({
  name: 'blog',
  entities: [entity({ config: Post })],
});
```

This generates five routes (`GET /posts`, `GET /posts/:id`, `POST /posts`, `PATCH /posts/:id`,
`DELETE /posts/:id`), all with input validation and OpenAPI documentation. No route handlers to
write.

## Adding operations

Operations model behavior beyond basic CRUD:

```typescript
import { defineOperations, op } from '@lastshotlabs/slingshot';

const PostOps = defineOperations(Post, {
  bySlug: op.lookup({ field: 'slug', unique: true }),
  publish: op.transition({
    field: 'status',
    from: 'draft',
    to: 'published',
    match: { id: 'param:id' },
  }),
});

// Then wire it in:
entity({ config: Post, operations: PostOps });
```

This generates `GET /posts/by-slug/:slug` and `POST /posts/:id/publish` automatically.

## Adding auth and permissions

Route policy controls auth, permissions, rate limits, and events per route:

```typescript
const Post = defineEntity('Post', {
  namespace: 'blog',
  fields: {
    /* ... */
  },
  routes: {
    defaults: { auth: 'userAuth' }, // all routes require login
    list: { auth: 'none' }, // except the list route
    create: { permission: 'blog:post.create' }, // create requires a specific permission
  },
});
```
