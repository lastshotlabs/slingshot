The entity system is how you author data-shaped domains the same way the framework's own packages
do.

It is code-first, package-friendly, and designed around generated behavior with explicit
customization seams.

## Use this when

Use the entity system when your package owns a data-shaped domain and you want:

- a canonical entity definition
- generated CRUD routes
- named operations beyond CRUD
- swappable storage backends
- route policy attached to the domain model

## The canonical flow

```typescript
import { defineOperations, definePackage, defineEntity, entity, field, op } from '@lastshotlabs/slingshot';

const Post = defineEntity('Post', {
  namespace: 'blog',
  fields: {
    id: field.string({ primary: true, default: 'uuid' }),
    authorId: field.string(),
    title: field.string(),
    status: field.enum(['draft', 'published'] as const, { default: 'draft' }),
  },
  routes: {
    defaults: { auth: 'userAuth' },
    list: { auth: 'none' },
  },
});

const PostOps = defineOperations(Post, {
  publish: op.transition({
    field: 'status',
    from: 'draft',
    to: 'published',
    match: { id: 'param:id' },
  }),
});

export const blogPackage = definePackage({
  name: 'blog',
  entities: [
    entity({
      config: Post,
      operations: PostOps,
    }),
  ],
});
```

## What the entity system gives you

- one canonical entity definition
- generated CRUD routes
- named operations beyond CRUD
- route policy for auth, permissions, rate limiting, idempotency, and events
- swappable storage backends
- customization seams for overrides and extra routes
- WebSocket channel declarations and other advanced wiring when needed

## Less common built-ins

These are still part of the canonical entity story and should be easy to discover:

- storage backend swapping
- consumer-controlled field and naming conventions
- factory-backed and composite adapter wiring
- generated route overrides
- extra routes inside the entity shell

## Escape hatches

When the generated path is not enough, the main escape hatches are:

- manual adapter wiring
- `createEntityPlugin()` for lower-level orchestration work
- generated-route executor overrides
- extra routes
- package routes outside the entity shell when the behavior is no longer entity-shaped
