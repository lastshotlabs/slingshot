These are the highest-signal starting points for the platform. They are organized around product and
authoring goals rather than package names so users can answer "which example looks like my app?"
quickly.

All examples use in-memory adapters by default. Swapping to SQLite, Postgres, MongoDB, or Redis is
a one-line change. See [Production Databases](/slingshot/examples/production-databases/).

Source-backed example apps live in the repo under `examples/`. The current source-backed set is
`with-auth`, `config-driven-domain`, `collaboration-workspace`, and `content-platform`.
