## Hello, world

```typescript title="app.config.ts"
import { defineApp, defineEntity, definePackage, entity, field } from '@lastshotlabs/slingshot';

const Task = defineEntity('Task', {
  namespace: 'app',
  fields: {
    id: field.string({ primary: true, default: 'uuid' }),
    title: field.string(),
    done: field.boolean({ default: false }),
  },
});

export default defineApp({
  port: 3000,
  packages: [definePackage({ name: 'app', entities: [entity({ config: Task })] })],
});
```

```bash
slingshot start
```

Five routes, request validation, OpenAPI docs, and swappable storage — from one definition.

## What you get

<CardGrid>
  <Card title="Generated CRUD" icon="seti:config">
    Define a model with `defineEntity`. Get `GET`, `POST`, `PATCH`, `DELETE` routes with validation
    and OpenAPI docs.
  </Card>
  <Card title="Operations" icon="seti:config">
    Add domain actions like `publish`, `archive`, `complete` alongside CRUD. Each generates a typed,
    documented route.
  </Card>
  <Card title="Events and the Event Bus" icon="comment-alt">
    One event triggers many effects — welcome email, search index, admin notification. No coupling
    between packages.
  </Card>
  <Card title="Realtime" icon="seti:html">
    SSE for live updates. WebSockets for chat and collaboration. Both built in.
  </Card>
  <Card title="Auth and Permissions" icon="seti:config">
    Sessions, OAuth, MFA, passkeys, RBAC, and resource-level access control.
  </Card>
  <Card title="Swappable Storage" icon="seti:config">
    Memory for dev, SQLite for prototyping, Postgres for production. Same entity code.
  </Card>
</CardGrid>
