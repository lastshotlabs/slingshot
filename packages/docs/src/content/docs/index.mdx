## Start with code

```typescript title="src/index.ts"
import { createServer, definePackage, domain, route } from '@lastshotlabs/slingshot';

const appPackage = definePackage({
  name: 'app',
  domains: [
    domain({
      name: 'health',
      basePath: '/health',
      routes: [
        route.get({
          path: '/',
          auth: 'none',
          handler: ({ respond }) => respond.json({ ok: true }),
        }),
      ],
    }),
  ],
});

await createServer({
  port: 3000,
  packages: [appPackage],
});
```

That is the main docs path:

- app root with `createServer()`
- package-first composition with `definePackage(...)`
- deeper domain authoring through the entity system when the package owns data-shaped behavior

Manifest authoring is still supported, but it now lives under [Alternate Paths](/slingshot/alternate-paths/).

## What Slingshot gives you by default

<CardGrid>
  <Card title="App Assembly" icon="seti:config">
    `createApp()` assembles the app and runtime context. `createServer()` adds the network server,
    workers, shutdown behavior, WebSockets, and SSE.
  </Card>
  <Card title="Events" icon="comment-alt">
    Every app gets an event bus. Typed event definitions drive canonical publishing, SSE, and other
    external delivery surfaces.
  </Card>
  <Card title="Realtime" icon="seti:html">
    WebSockets and SSE are both core, but they are different transports with different scaling and
    authoring concerns.
  </Card>
  <Card title="Generated Docs" icon="seti:config">
    Validation and OpenAPI are part of the normal authoring path, not an afterthought.
  </Card>
  <Card title="Swappable Storage" icon="seti:config">
    The entity system can target memory, Redis, SQLite, MongoDB, and Postgres from one domain model.
  </Card>
  <Card title="Escape Hatches" icon="seti:config">
    Raw plugins, manual adapter wiring, generated route overrides, and custom routes are all still
    available when you need them.
  </Card>
</CardGrid>

## Built-in product surface

Slingshot also ships larger product packages for auth, community, chat, organizations, search,
rendering, assets, and integrations. Those packages are important, but the docs now start with the
core framework model they are built on.
