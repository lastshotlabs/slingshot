## Get a full backend running in seconds

<Tabs>
  <TabItem label="Manifest (zero-code)">
    ```json title="app.manifest.json"
    {
      "manifestVersion": 1,
      "port": 3000,
      "plugins": [
        { "plugin": "slingshot-auth" },
        { "plugin": "slingshot-community" }
      ]
    }
    ```

    ```bash
    slingshot start
    ```

  </TabItem>
  <TabItem label="Code">
    ```typescript title="src/index.ts"
    import { createServer } from '@lastshotlabs/slingshot';
    import { createAuthPlugin } from '@lastshotlabs/slingshot-auth';
    import { createCommunityPlugin } from '@lastshotlabs/slingshot-community';

    await createServer({
      port: 3000,
      security: { signing: { secret: process.env.SECRET! } },
      plugins: [
        createAuthPlugin({
          auth: { roles: ['user'], defaultRole: 'user' },
          db: { auth: 'memory', sessions: 'memory', oauthState: 'memory' },
        }),
        createCommunityPlugin({ containerCreation: 'user' }),
      ],
    });
    ```

  </TabItem>
</Tabs>

Either way, you get the framework assembly layer plus a broader plugin ecosystem: auth and
identity, community and chat, organizations and permissions, search, SSR/SSG, assets and media,
and integration plugins such as webhooks, mail, and push.

## Adapters swap without touching your logic

Start with in-memory storage. Switch to SQLite, Postgres, MongoDB, or Redis with a one-line change - nothing else changes.

```json
"plugins": [{
  "plugin": "slingshot-auth",
  "config": { "db": { "auth": "sqlite", "sessions": "redis" } }
}]
```

## Define your own domains the same way

The platform packages aren't special. Any domain can work this way:

```typescript title="src/entities/post.ts"
export const Post = defineEntity('Post', {
  namespace: 'blog',
  fields: {
    id: field.string({ primary: true, default: 'uuid' }),
    authorId: field.string(),
    title: field.string(),
    body: field.string(),
    status: field.enum(['draft', 'published'] as const, { default: 'draft' }),
  },
  routes: {
    defaults: { auth: 'userAuth' },
    get: { auth: 'none' },
    list: { auth: 'none' },
    create: {
      permission: { requires: 'blog:post.write' },
      event: { key: 'blog:post.created', payload: ['id', 'authorId', 'title'] },
    },
  },
});
```

One definition. CRUD routes, a typed adapter for every supported database, permission enforcement, and event emission - all wired automatically.
