Slingshot reads secrets at startup, not at request time. Missing or misconfigured secrets fail loud and fast — before the server serves a single request.

## How secrets are provided

Configure a provider in the manifest:

```json
{
  "secrets": {
    "provider": "env",
    "prefix": "APP_"
  }
}
```

Or in code:

```typescript
await createServer({
  secrets: {
    provider: 'env',
    prefix: 'APP_',
  },
});
```

## Providers

### `env` — environment variables

Reads from `process.env`. The `prefix` option filters to matching variables only:

```json
{ "provider": "env", "prefix": "APP_" }
```

`APP_SECRET` becomes `SECRET`, `APP_DB_URL` becomes `DB_URL`, and so on.

### `file` — `.env` file

Reads a `.env` file and populates `process.env` before the app boots:

```json
{ "provider": "file", "path": "${importMetaDir}/.env" }
```

### `ssm` — AWS SSM Parameter Store

Fetches from SSM at startup:

```json
{
  "provider": "ssm",
  "path": "/my-app/prod/",
  "region": "us-east-1"
}
```

Parameters under `/my-app/prod/` are available by their trailing key name. `/my-app/prod/SECRET` becomes `SECRET`.

## What uses secrets

The most common usage is `signing.secret` for JWT signing:

```json
{
  "security": {
    "signing": {
      "secret": "${secret:SIGNING_SECRET}"
    }
  }
}
```

`${secret:NAME}` resolves via the configured provider at startup.

## Fail fast

Missing secrets throw at startup:

```
Error: Required secret SIGNING_SECRET not found in environment.
Make sure APP_SIGNING_SECRET is set (using prefix "APP_").
```

A server that boots has all its secrets. Nothing resolves lazily at request time.

## Custom secrets provider

Register one via the handler registry:

```typescript
const registry = createManifestHandlerRegistry();
registry.registerSecretsProvider('vault', config => {
  return {
    async get(key: string): Promise<string | undefined> {
      return vaultClient.read(`${config.path}${key}`);
    },
  };
});

await createServerFromManifest('./app.manifest.json', registry);
```

Then reference it by name in the manifest:

```json
{
  "secrets": {
    "provider": "vault",
    "path": "secret/data/my-app/"
  }
}
```
