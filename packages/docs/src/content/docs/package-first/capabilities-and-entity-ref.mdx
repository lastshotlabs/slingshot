---
title: Capabilities and entityRef
description: Typed cross-package contracts for services and entity adapters
---

Capabilities and entity refs are how packages depend on each other without hard-coding adapter bags
or global singletons.

These examples extend the [starter app](/slingshot/app-authoring/starter-app/) with a mail package
and a search package that read from `blogPackage`.

## Capabilities

Use a capability when one package wants to publish a typed service for another package to consume.

```typescript title="src/mail/package.ts"
import {
  defineCapability,
  definePackage,
  domain,
  provideCapability,
  route,
} from '@lastshotlabs/slingshot';

const Mailer = defineCapability<{ send(to: string, body: string): Promise<void> }>('mailer');

const mailPackage = definePackage({
  name: 'mail',
  capabilities: {
    provides: [
      provideCapability(Mailer, () => ({
        send: async (to, body) => {
          console.log('sending mail', { to, body });
        },
      })),
    ],
  },
});

const notificationsPackage = definePackage({
  name: 'notifications',
  dependencies: ['mail'],
  capabilities: {
    requires: [Mailer],
  },
  domains: [
    domain({
      name: 'notifications',
      routes: [
        route.post({
          path: '/notify',
          auth: 'none',
          handler: async ({ capabilities, respond }) => {
            await capabilities.require(Mailer).send('user@example.com', 'A post was published');
            return respond.noContent();
          },
        }),
      ],
    }),
  ],
});
```

## `entityRef(...)`

Use an entity ref when one package needs a typed handle to another package's entity adapter.

```typescript title="src/search/package.ts"
import { definePackage, domain, entityRef, route } from '@lastshotlabs/slingshot';
import type { PackageEntityAdapterFor } from '@lastshotlabs/slingshot-entity';
import { Post } from '../blog/post';
import { PostOperations } from '../blog/post-operations';

type PostAdapter = PackageEntityAdapterFor<typeof Post, typeof PostOperations.operations>;

const PostRef = entityRef<PostAdapter>({
  plugin: 'blog',
  entity: 'Post',
});

export const searchPackage = definePackage({
  name: 'search',
  dependencies: ['blog'],
  domains: [
    domain({
      name: 'search',
      basePath: '/search',
      routes: [
        route.get({
          path: '/posts/by-slug/:slug',
          auth: 'none',
          handler: async ({ entities, request, respond }) => {
            const posts = entities.get(PostRef);
            const result = await posts.bySlug({ slug: request.req.param('slug') });
            return respond.json({ result });
          },
        }),
      ],
    }),
  ],
});
```

Then package routes can resolve the adapter through `ctx.entities.get(...)` without coupling the
packages through direct imports of runtime state.

## Why these are separate concepts

- capabilities are for service-like contracts
- entity refs are for framework-managed entity adapters

That split keeps cross-package contracts explicit.

## When to use them

Use a capability when the dependency is behavior.

Use an entity ref when the dependency is data access through a framework-managed entity surface.
