title: Escape Hatches
description: When to drop below the package surface — raw plugins, executor overrides, manual wiring

Package-first authoring covers most apps. When it does not fit, the framework has escape
hatches at every level.

## Decision guide

| You need...                                          | Escape hatch                 |
| ---------------------------------------------------- | ---------------------------- |
| Custom logic in a generated CRUD route               | Executor override            |
| A new route inside the entity shell                  | Extra route                  |
| Full control over entity storage construction        | Manual adapter wiring        |
| Direct lifecycle control (middleware, teardown)      | Raw plugin                   |
| Custom system field names for an existing database   | Entity `systemFields` config |
| Custom storage conventions (Redis keys, table names) | Entity `conventions` config  |

## Executor overrides

Replace the handler behind a generated route without changing its identity:

```typescript
// @skip-typecheck
import { defineEntityExecutor, entity } from '@lastshotlabs/slingshot-entity';

entity({
  config: Post,
  operations: PostOps,
  overrides: {
    get: defineEntityExecutor({
      summary: 'Get post with view tracking',
      build:
        ({ entityAdapter }) =>
        async ({ params, respond }) => {
          const post = await entityAdapter.get(params.id);
          if (!post) return respond.notFound();
          // Track the view
          await entityAdapter.update(params.id, { views: (post.views ?? 0) + 1 });
          return respond.json(post);
        },
    }),
  },
});
```

See [Generated Routes, Overrides, and Extra Routes](/slingshot/entity-system/generated-routes-overrides-and-extra-routes/) for full details.

## Extra routes

Add routes that live alongside generated entity routes:

```typescript
// @skip-typecheck
import { defineEntityRoute, entity } from '@lastshotlabs/slingshot-entity';

entity({
  config: Post,
  extraRoutes: [
    defineEntityRoute({
      key: 'archive-all',
      method: 'post',
      path: '/archive-all',
      auth: 'userAuth',
      permission: { requires: 'blog:post.moderate' },
      buildExecutor:
        ({ entityAdapter }) =>
        async ({ respond }) => {
          // Custom batch operation
          const all = await entityAdapter.list({});
          for (const post of all.items) {
            await entityAdapter.update(post.id, { status: 'archived' });
          }
          return respond.json({ archived: all.items.length });
        },
    }),
  ],
});
```

## Manual adapter wiring

Build your own storage adapter when the generated ones do not fit:

```typescript
// @skip-typecheck
entity({
  config: Post,
  wiring: {
    mode: 'manual',
    buildAdapter: (storeType, infra) => {
      // Use your own ORM, raw SQL, or external API
      return {
        get: async id => {
          /* ... */
        },
        list: async opts => {
          /* ... */
        },
        create: async data => {
          /* ... */
        },
        update: async (id, data) => {
          /* ... */
        },
        delete: async id => {
          /* ... */
        },
      };
    },
  },
});
```

See [Storage and Adapter Wiring](/slingshot/entity-system/storage-and-adapter-wiring/) for the factories and manual wiring modes.

## Raw plugins

Use a raw plugin when you need direct lifecycle control:

```typescript
// @skip-typecheck
import type { SlingshotPlugin } from '@lastshotlabs/slingshot-core';

const metricsPlugin: SlingshotPlugin = {
  name: 'metrics',
  setupMiddleware({ app }) {
    app.use('*', async (c, next) => {
      const start = Date.now();
      await next();
      const ms = Date.now() - start;
      console.log(`${c.req.method} ${c.req.path} ${c.res.status} ${ms}ms`);
    });
  },
  setupPost({ ctx }) {
    console.log(`Metrics plugin ready. ${ctx.plugins.length} plugins loaded.`);
  },
  teardown() {
    console.log('Metrics plugin shutting down');
  },
};
```

Register it in the app root:

```typescript title="app.config.ts"
// @skip-typecheck
import { defineApp } from '@lastshotlabs/slingshot';

export default defineApp({
  port: 3000,
  plugins: [metricsPlugin],
  packages: [blogPackage],
});
```

### Plugin lifecycle hooks

| Hook              | When it runs                                   |
| ----------------- | ---------------------------------------------- |
| `setupMiddleware` | During middleware registration (before routes) |
| `setupRoutes`     | During route registration                      |
| `setupPost`       | After all routes are registered                |
| `seed`            | During seed phase (for initial data setup)     |
| `teardown`        | During graceful shutdown                       |

## Entity system field conventions

Override generated field names to match an existing database schema:

```typescript
const LegacyPost = defineEntity('LegacyPost', {
  namespace: 'blog',
  fields: {
    id: field.string({ primary: true }),
    title: field.string(),
  },
  systemFields: {
    ownerField: 'author_id', // not 'authorId'
    tenantField: 'organization_id', // not 'orgId'
  },
  storageFields: {
    ttlField: '_expires_at',
  },
  storage: {
    postgres: { tableName: 'blog_posts_v2' },
    redis: { keyPrefix: 'bp' },
  },
  conventions: {
    redisKey: ({ storageName, id }) => `${storageName}:v2:${id}`,
  },
});
```

## The rule

Stay on the package surface until you have a concrete reason to leave it. The escape hatches
exist so you never hit a wall — not so you start at the lowest level.

Typical progression:

1. **Start with packages, entities, and route policy** — covers 90% of apps
2. **Add overrides or extra routes** — when generated behavior needs tweaking
3. **Use capabilities and entityRef** — when packages need to talk to each other
4. **Drop to raw plugins** — only for infrastructure concerns like metrics, tracing, or custom middleware registration
