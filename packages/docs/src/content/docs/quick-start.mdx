## Install

```bash
bun add @lastshotlabs/slingshot zod
```

## Step 1: A route that works

Create `src/index.ts`:

```typescript title="src/index.ts"
import { createServer, definePackage, domain, route } from '@lastshotlabs/slingshot';

const appPackage = definePackage({
  name: 'app',
  domains: [
    domain({
      name: 'health',
      basePath: '/health',
      routes: [
        route.get({
          path: '/',
          auth: 'none',
          summary: 'Health check',
          handler: ({ respond }) => respond.json({ ok: true }),
        }),
      ],
    }),
  ],
});

await createServer({
  port: 3000,
  packages: [appPackage],
});
```

Run it:

```bash
bun run src/index.ts
```

Test it:

```bash
curl http://localhost:3000/health
# → { "ok": true }
```

Your API docs are already live at `http://localhost:3000/docs`.

**What just happened:** `createServer` started a Hono server. `definePackage` created a module
with one route. The route is typed, shows up in OpenAPI docs, and can be tested with `curl`.

## Step 2: Add a data model

Most backends need CRUD for a data model. Instead of writing five routes by hand, define an entity:

```typescript title="src/entities/post.ts"
import { defineEntity, field } from '@lastshotlabs/slingshot';

export const Post = defineEntity('Post', {
  namespace: 'blog',
  fields: {
    id: field.string({ primary: true, default: 'uuid' }),
    title: field.string(),
    body: field.string(),
  },
});
```

Wire it into your package:

```typescript title="src/index.ts"
import { createServer, definePackage, entity } from '@lastshotlabs/slingshot';
import { Post } from './entities/post';

const blogPackage = definePackage({
  name: 'blog',
  entities: [entity({ config: Post })],
});

await createServer({
  port: 3000,
  packages: [blogPackage],
});
```

That's it. You now have:

```bash
curl -X POST http://localhost:3000/posts \
  -H "Content-Type: application/json" \
  -d '{ "title": "Hello", "body": "World" }'
# → { "id": "01J...", "title": "Hello", "body": "World" }

curl http://localhost:3000/posts
# → [{ "id": "01J...", "title": "Hello", "body": "World" }]
```

Five routes generated from one definition: `GET /posts`, `GET /posts/:id`, `POST /posts`,
`PATCH /posts/:id`, `DELETE /posts/:id`. All with input validation and OpenAPI docs.

## Step 3: Add auth to some routes

Most apps need some routes to require login. Add route defaults:

```typescript title="src/entities/post.ts"
export const Post = defineEntity('Post', {
  namespace: 'blog',
  fields: {
    id: field.string({ primary: true, default: 'uuid' }),
    title: field.string(),
    body: field.string(),
  },
  routes: {
    defaults: { auth: 'userAuth' }, // all routes require login by default
    list: { auth: 'none' }, // anyone can list posts
    get: { auth: 'none' }, // anyone can read a post
  },
});
```

Now `POST`, `PATCH`, and `DELETE` require a valid JWT. `GET` routes are public.

To actually issue JWTs, add the auth package — see [Auth Setup](/slingshot/examples/with-auth/).
