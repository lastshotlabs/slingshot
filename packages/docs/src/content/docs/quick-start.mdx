## Install

```bash
bun add @lastshotlabs/slingshot zod
```

## Start with one package and one route

Create `src/index.ts`:

```typescript title="src/index.ts"
import { createServer, definePackage, domain, route } from '@lastshotlabs/slingshot';

const appPackage = definePackage({
  name: 'app',
  domains: [
    domain({
      name: 'health',
      basePath: '/health',
      routes: [
        route.get({
          path: '/',
          auth: 'none',
          summary: 'Health check',
          handler: ({ respond }) => respond.json({ ok: true }),
        }),
      ],
    }),
  ],
});

await createServer({
  port: 3000,
  packages: [appPackage],
});
```

Run it:

```bash
bun run src/index.ts
```

Now hit:

```bash
curl http://localhost:3000/health
```

You should get:

```json
{ "ok": true }
```

Your generated docs are also live at `http://localhost:3000/docs`.

## This is the canonical scaffold

If you are asking "what is the normal Slingshot app shape?", it is this:

- `createServer(...)` at the app root
- `packages: [...]` as the main composition surface
- `definePackage(...)` for package modules
- `defineEntity(...)` when a package owns a data-shaped domain

## Add an entity

When the package owns a data-shaped domain, move to the entity system.

```typescript title="src/post.ts"
import { defineEntity, field } from '@lastshotlabs/slingshot';

export const Post = defineEntity('Post', {
  namespace: 'blog',
  fields: {
    id: field.string({ primary: true, default: 'uuid' }),
    authorId: field.string(),
    title: field.string(),
    body: field.string(),
  },
  routes: {
    defaults: { auth: 'userAuth' },
    list: { auth: 'none' },
    get: { auth: 'none' },
  },
});
```

That is the canonical next step for most real apps.
