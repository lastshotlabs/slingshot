Realtime is now documented as two separate first-class core features:

- [WebSockets](/slingshot/app-authoring/websockets/)
- [Server-Sent Events](/slingshot/app-authoring/server-sent-events/)

That split is intentional. They are different transports with different runtime models, scaling
concerns, and authoring patterns.

## Use this page as a chooser

| If you need... | Go here |
| --- | --- |
| Bidirectional communication, rooms, presence, recovery, client-to-server events | [WebSockets](/slingshot/app-authoring/websockets/) |
| Server-to-client event streams for browsers | [Server-Sent Events](/slingshot/app-authoring/server-sent-events/) |
| The shared event model behind SSE and other consumers | [Events and the Event Bus](/slingshot/app-authoring/events-and-the-event-bus/) |
| Multi-instance delivery and operational scaling | [Horizontal Scaling](/slingshot/guides/horizontal-scaling/) |

## Recommended order

1. [Events and the Event Bus](/slingshot/app-authoring/events-and-the-event-bus/)
2. [WebSockets](/slingshot/app-authoring/websockets/) or [Server-Sent Events](/slingshot/app-authoring/server-sent-events/)
3. [Examples](/slingshot/examples/) for a product-shaped reference
